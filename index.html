Overview

This application is a currency exchange platform designed to provide users with real-time conversion rates between USD and MXN (Mexican Peso) and USD and COP (Colombian Peso). It allows users to input an amount in USD, view the equivalent amount in their chosen currency, and confirm the transaction. Once confirmed, the transaction details are securely stored for future reference.

Key Features

  • Real-Time Currency Conversion: The application provides up-to-date conversion rates for USD to MXN and USD to COP, allowing users to quickly see the equivalent amount in their preferred currency.
  • Transaction Confirmation: Users can confirm a transaction, which triggers secure storage of the transaction details in the backend. This includes the amount converted, exchange rate used, and other relevant transaction details.
  • Scalable Serverless Architecture: Built using AWS serverless components, the application is highly scalable and optimized for performance, allowing it to handle large volumes of requests with minimal latency.
  • Caching for Fast Response: Exchange rates are cached to ensure quick access and minimal load on the database. This improves the user experience by reducing response times.
  • Secure Data Handling: Sensitive information, such as transaction records and API keys, is stored and managed securely using AWS Secrets Manager and Parameter Store, following best practices for security and data privacy.

Technology Stack

  • Frontend: Built with React, the frontend is hosted on Amazon S3 for scalable static storage, with Amazon CloudFront as a Content Delivery Network (CDN) to ensure low-latency delivery globally. React enables a dynamic and responsive user interface for real-time currency conversions.
  • Backend: The backend logic is implemented in Python, using AWS Lambda to handle core functionalities like currency conversion calculations and transaction processing. Python’s integration with AWS Lambda allows for fast, scalable, and efficient processing of backend tasks.
  • API Management: Amazon API Gateway serves as the entry point for backend requests, managing and routing API calls to the appropriate Python Lambda functions.
  • Request Queueing: Amazon SQS (Simple Queue Service) queues incoming requests to handle high traffic and ensure smooth processing during peak times.
  • Database: Amazon RDS is used for storing confirmed transaction records, ensuring data persistence and durability.
  • Caching: Amazon ElastiCache (Redis) is used to cache exchange rates, minimizing response times for frequent requests.
  • Security: AWS Secrets Manager and Parameter Store manage secure access to configuration settings and sensitive information.
